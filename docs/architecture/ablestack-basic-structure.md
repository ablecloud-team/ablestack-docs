    <article class="markdown-body entry-content container-lg" itemprop="text"><p>ABLESTACK은 전통적인 데이터센터의 문제점을 해결하여 새로운 개념의 데이터센터를 기업에 제공하기 위해 x86 기반의 일반 상용(Off-the-shelf)서버와 다양한 소프트웨어를 결합합니다. 이러한 결합을 위해서는 단순히 하드웨어에 소프트웨어를 설치하는 것이 아닌 ABLESTACK이 설계한 최적의 아키텍처가 적용되야 합니다. 본 문서에서는 ABLESTACK의 상세한 아키텍처를 살펴보기 전에 전체적인 구성요소와 구성요소의 역할에 대해 설명하고자 합니다.</p>
<p>ABLESTACK은 다양한 기술을 적용해서 상용 서버에 적용하기 위해 크게 5개의 구성요소로 이루어져 있습니다. 다음과 같습니다.</p>
<ul>
<li>ABLESTACK
<ul>
<li>Wall : 인프라 전체에 대한 모니터링, 알람 등을 위한 통합 모니터링 플랫폼</li>
<li>Mold : 가상머신 관리 및 소프트웨어 정의 네트워크, 오케스트레이션 및 오토메이션을 제공하는 클라우드 플랫폼</li>
<li>Cell : 서버 가상화를 지원하는 가상화 하이퍼바이저</li>
<li>Glue : 소프트웨어 정의 스토리지를 통해 통합 스토리지를 제공하고, 다양한 게이트웨이를 제공하는 스토리지 플랫폼</li>
<li>Cube : x86 기반의 서버에 Linux Kernel을 제공하고, 호스트 관리 환경을 제공하는 서버 OS</li>
</ul>
</li>
</ul>
<p>다음의 그림은 ABLESTACK HCI 플랫폼의 전체적인 구조를 보여줍니다.</p>

![ablestack-platform-architecture](../assets/images/ablestack-platform-architecture.png)

<p>각각의 구성요소의 주요 내용과 특징은 다음과 같습니다.</p>
<h2><a id="user-content-cube--호스트용-linux-os" class="anchor" aria-hidden="true" href="#cube--호스트용-linux-os"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Cube : 호스트용 Linux OS</h2>
<p>Cube는 ABLESTACK 클러스터를 구성하는 호스트에 설치하기 위한 Linux OS 입니다. 설치 미디어를 호스트에 장착하고, 설치를 시작하면 GUI 형태의 설치 마법사를 이용해 바로 OS를 설치할 수 있습니다. 이 설치 과정에서 ABLESTACK을 구동하기 위한 소프트웨어가 자동으로 설치됩니다.</p>
<p>ABLESTACK Cube는 다음과 같은 주요 기능과 특징을 가집니다.</p>
<h3><a id="user-content-downstream-linux-os" class="anchor" aria-hidden="true" href="#downstream-linux-os"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Downstream Linux OS</h3>
<p>Cube는 엔터프라이즈 환경에서 안정적인 운영 환경을 제공하기 위해 Enterprise Linux OS의 다운스트림인 CentOS를 기반으로 합니다. 다운스트림 OS란 상용으로 릴리즈 되어 엔터프라이즈 환경에서 운영되는 제품의 형상을 그대로 사용하여 배포되는 OS를 의미합니다.</p>
<p>그만큼 매우 안정적입니다. 또한 업스트림에서 발생한 이슈가 다운스트림에 자동으로 적용되기 때문에 언제든 최적의 상태를 사용자에게 제공할 수 있습니다.</p>
<h3><a id="user-content-웹-기반-호스트-관리-기능-제공" class="anchor" aria-hidden="true" href="#웹-기반-호스트-관리-기능-제공"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>웹 기반 호스트 관리 기능 제공</h3>
<p>Cube를 호스트에 설치하면 바로 웹 기반의 호스트 관리 기능을 사용할 수 있습니다. 웹 브라우저를 통해 실행되기 때문에 어떤 하드웨어에서도 기능을 사용할 수 있습니다. 호스트에 대한 기본적인 정보 조회, 호스트의 디스크 장치 및 네트워크 장치에 대한 관리, 호스트에서 실행 중인 가상머신 관리, 서비스 관리, ABLESTACK 관리 기능 등을 웹에서 모두 사용할 수 있습니다.</p>
<h3><a id="user-content-ablestack-배포-마법사" class="anchor" aria-hidden="true" href="#ablestack-배포-마법사"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>ABLESTACK 배포 마법사</h3>

<p>Mold의 가장 강력한 기능은 자유롭게 다양한 네트워크를 생성하고, 모든 기능을 웹에서 컨트롤 할 수 있다는 것입니다. 라우터를 제공하며, 무수한 서브넷을 생성할 수 있고, 라우팅 및 방화벽, NAT, VPN, Security Group 등을 생성하고 관리함으로써 외부 및 가상머신 간의 트래픽을 효과적으로 통제할 수 있습니다. 사용자별로 격리된 네트워크를 제공하고, 가상머신 간의 네트워크 흐름을 완벽하게 통제함으로써 마이크로세그멘테이션을 제공하여 안전한 가상머신 운영을 보장합니다.</p>
<h3><a id="user-content-자동화-및-오케스트레이션" class="anchor" aria-hidden="true" href="#자동화-및-오케스트레이션"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>자동화 및 오케스트레이션</h3>
<p>Mold는 강력한 자동화 및 오케스트레이션 기능을 내장하고 있습니다. 사용자는 빠르게 가상머신을 배포하고 애플리케이션을 설정할 수 있습니다. 미리 정의된 규칙에 의해 가상머신을 생성하고 관리하기 때문에 시스템의 일관성을 유지할 수 있어 관리 효율성을 높일 수 있습니다.</p>
<h3><a id="user-content-가상머신-및-컨테이너-지원" class="anchor" aria-hidden="true" href="#가상머신-및-컨테이너-지원"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>가상머신 및 컨테이너 지원</h3>
<p>Mold는 가상머신 뿐 아니라 최신 가상화 기술인 컨테이너를 지원합니다. Kubernetes 클러스터를 자동으로 배포하고, 관리화면을 통해 컨테이너로 만들어진 애플리케이션을 바로 배포하고 실행할 수 있습니다. 사용자는 Mold를 통해 어떠한 형태의 애플리케이션이든 실행할 수 있습니다.</p>
<h2><a id="user-content-wall--모니터링-플랫폼" class="anchor" aria-hidden="true" href="#wall--모니터링-플랫폼"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Wall : 모니터링 플랫폼</h2>
<p>Wall은 ABLESTACK의 강력한 모니터링 플랫폼입니다. 호스트 및 가상머신에 대한 실시간 상태를 확인할 수 있고 기간별 인사이트를 제공합니다.</p>
<h3><a id="user-content-강력한-실시간-시각화" class="anchor" aria-hidden="true" href="#강력한-실시간-시각화"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>강력한 실시간 시각화</h3>
<p>호스트 및 가상머신 등의 자원에 대한 강력한 실시간 시각화를 제공합니다. 사용자는 현재 상태의 호스트와 가상머신에 대한 실시간 그래프를 보며 상태를 조회할 수 있어 효율적인 모니터링이 가능합니다.</p>
<h3><a id="user-content-기간별-인사이트-제공" class="anchor" aria-hidden="true" href="#기간별-인사이트-제공"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>기간별 인사이트 제공</h3>
<p>호스트 및 가상머신 등의 자원에 대한 모니터링 정보를 기간별로 선택하여 분석할 수 있으며 언제든지 파일로 다운로드 할 수 있습니다.</p>
<h3><a id="user-content-인공지능-분석" class="anchor" aria-hidden="true" href="#인공지능-분석"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>인공지능 분석</h3>
<p>인공지능을 이용해 호스트 및 가상머신의 모니터링 정보를 분석/학습하여 사용량을 예측하고, 데이터를 제공함으로써 시스템 확장을 적절하게 미리 준비할 수 있습니다.</p>
